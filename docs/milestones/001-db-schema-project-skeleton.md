# Milestone 001 — DB Schema + Project Skeleton

## 1. Goal

Создать работающий скелет FastAPI-проекта с:
- SQLAlchemy моделями (users, jobs, usage_log)
- Alembic миграциями (initial schema)
- RQ/Redis connection skeleton
- PostgreSQL connection
- GET /health endpoint
- pytest smoke tests
- config/env loading без hardcoded secrets

После этого milestone: проект запускается, `/health` возвращает статус БД и Redis,
`alembic upgrade head` проходит, `pytest` зелёный.

---

## 2. In Scope

- Базовая структура FastAPI-проекта (`app/`, `alembic/`, `tests/`)
- Config/env loading через pydantic-settings (`.env` / environment variables)
- SQLAlchemy 2.x models:
  - `users` (id, email nullable unique, tier default "free", created_at)
  - `jobs` (id, user_id nullable FK, session_id, status enum, original_filename,
    stored_filename, input_path NOT NULL, output_path nullable,
    video_size_bytes, duration_seconds nullable,
    created_at, completed_at nullable, expires_at, error_message nullable)
  - `usage_log` (id, job_id FK, user_id nullable FK, session_id nullable,
    minutes_processed, created_at)
- Alembic init + `alembic.ini`
- Initial migration (создаёт все три таблицы)
- Redis connection skeleton (rq + redis-py, проверка ping)
- PostgreSQL connection (psycopg2, синхронный — совместимость с Alembic)
- `GET /health` → `{"status": "ok", "db": "ok", "redis": "ok"}`
- pytest smoke tests (health endpoint + DB connectivity)
- `.env.example` (только placeholder values, без реальных секретов)
- `requirements.txt`
- `Dockerfile`
- `docker-compose.yml` (app + postgres:15 + redis:7)
- Обновить `docs/agents/backend-agent.md` (убрать устаревшее "No users table in MVP")

## 8. DB Schema Reference

### `users`
| Column | Type | Constraints |
|---|---|---|
| id | UUID | PK, generated by application (uuid.uuid4) |
| email | VARCHAR | nullable, unique |
| tier | VARCHAR | NOT NULL, default 'free' |
| created_at | TIMESTAMP | NOT NULL, default now() |

### `jobs`
| Column | Type | Constraints |
|---|---|---|
| id | UUID | PK |
| user_id | UUID | nullable, FK → users.id |
| session_id | VARCHAR | NOT NULL |
| status | ENUM | pending/queued/processing/done/failed |
| original_filename | VARCHAR | NOT NULL |
| stored_filename | VARCHAR | NOT NULL |
| input_path | VARCHAR | NOT NULL — путь к сохранённому входному видео |
| output_path | VARCHAR | nullable — путь к готовому ZIP после обработки |
| video_size_bytes | BIGINT | nullable |
| duration_seconds | FLOAT | nullable |
| created_at | TIMESTAMP | NOT NULL |
| completed_at | TIMESTAMP | nullable |
| expires_at | TIMESTAMP | NOT NULL |
| error_message | TEXT | nullable |

### `usage_log`
| Column | Type | Constraints |
|---|---|---|
| id | UUID | PK |
| job_id | UUID | NOT NULL, FK → jobs.id |
| user_id | UUID | nullable, FK → users.id |
| session_id | VARCHAR | nullable |
| minutes_processed | NUMERIC(10,2) | nullable |
| created_at | TIMESTAMP | NOT NULL |

## 9. Verification Steps

### DB Connectivity Gate (Docker-based, canonical)

Real DB connectivity is verified via Docker commands — **not pytest**.
pytest tests are unit smoke tests with mocked DB and do not test real connectivity.

```powershell
# 1. Apply migrations inside container
docker compose run --rm app python -m alembic upgrade head

# 2. Start the app
docker compose up -d app

# 3. Check health endpoint
Invoke-RestMethod http://localhost:8000/health
# Expected: status=ok, db=ok, redis=ok
```

### Full Local Verification

```bash
# 1. Поднять инфраструктуру
docker compose up -d postgres redis

# 2. Применить миграции (через Docker)
docker compose run --rm app python -m alembic upgrade head

# 3. Запустить приложение
docker compose up -d app

# 4. Проверить health endpoint
curl http://localhost:8000/health
# Expected: {"status": "ok", "db": "ok", "redis": "ok"}

# 5. Запустить тесты (unit smoke tests, mocked DB)
pytest tests/ -v

# 6. Проверить отсутствие секретов
grep -r "password\|secret\|api_key" app/ --include="*.py" | grep -v ".env"
```
